# Shiki → Obsidian (PowerShell)

Скрипт синхронизирует ваш список аниме с [Shikimori](https://shikimori.io) (или совместимого инстанса API): подтягивает OAuth-токены, все оценки пользователя, детали тайтлов с дисковым кэшем, генерирует Markdown-заметки с YAML front matter и скачивает обложки в хранилище Obsidian.

## Требования

- **PowerShell 7+** (`pwsh`) — рекомендуется: в `shiki.notes.psm1` используется тернарный оператор (`? :`), которого нет в Windows PowerShell 5.1.
- Файл **`vault.path`** в корне проекта: одна строка — абсолютный путь к корню хранилища Obsidian.
- **`shiki_tokens.json`** — OAuth-токены приложения Shikimori (см. ниже).
- **`shiki.config.psd1`** — настройки путей и API (см. ниже).

## Запуск

```powershell
cd "d:\Desktop\Projects\shiki_powershell"
pwsh -File .\shiki.main.ps1
```

Транскрипт пишется в `shiki-sync.log` (или путь из `TranscriptPath` в конфиге).

## Конфигурация (`shiki.config.psd1`)

| Ключ | Назначение |
|------|------------|
| `NotesRel` | Папка заметок относительно корня vault (например `01 - Base/05 - Anime`). Если пусто — в коде заданы запасные значения. |
| `PostersRel` | Папка для JPG постеров относительно vault. |
| `UA` | User-Agent для HTTP и API. |
| `SiteUrl` | **Основной URL сайта и API** (без завершающего `/`). Меняйте при смене домена, например `https://shikimori.io`. Внутри скрипта попадает в `Base`. |
| `StaticUrl` | База CDN для обложек (по умолчанию `https://desu.shikimori.one`). Внутри — `StaticBase`. |
| `Base` / `StaticBase` | Устаревшие имена; если заданы без `SiteUrl`/`StaticUrl`, всё ещё читаются для совместимости. |
| `PosterThrottle` | Параллельность загрузки постеров (только при PS7+, `ForEach-Object -Parallel`). |
| `MaxScore` | Верхняя граница при разборе оценки из комментария к тайтлу. |
| `ShikiCookie` | Необязательно: заголовок Cookie для скачивания постеров, если CDN требует сессию. |
| `MinPosterBytes` | Минимальный размер файла, чтобы считать постер «достаточным»; иначе берётся лучший из кандидатов. |
| `CacheBust` | Добавлять `?t=…` к URL при скачивании постеров. |

Опциональные переопределения путей (если заданы в хэштаблице конфига):

- `SecretPath` — вместо `shiki_tokens.json` в корне проекта.
- `TranscriptPath` — лог транскрипта.
- `CacheDir` — каталог JSON-кэша деталей аниме (по умолчанию `shiki_cache`).

**Замечание:** в конфиге есть `DetailThrottle`, `BatchSize` — в текущей версии `shiki.main.ps1` для пакетов деталей зашиты `-BatchSize 25` и `-Throttle 1`; при изменении логики синхронизации имеет смысл выровнять с конфигом.

## Токены (`shiki_tokens.json`)

JSON с полями OAuth (используются в `shiki.oauth.psm1`): `client_id`, `client_secret`, `refresh_token`, `access_token`, `expires_at`. При запуске при необходимости выполняется refresh и файл перезаписывается.

**Не публикуйте** этот файл и `vault.path` в открытых репозиториях.

## Структура проекта

| Файл | Роль |
|------|------|
| `shiki.main.ps1` | Точка входа: токены, профиль, список оценок, детали, заметки, постеры. |
| `shiki.oauth.psm1` | Refresh токена, `Get-ShikiTokens`, заголовки. |
| `shiki.shiki.psm1` | API: профиль, все оценки, детали по ID с кэшем на диске, retry при 429/503. |
| `shiki.notes.psm1` | Генерация Markdown с YAML: теги, оценка из комментария или из поля, жанры, студии, рейтинг аудитории. |
| `shiki.http.psm1` | `Download-PosterFromCandidates` — перебор URL, минимальный размер, cache-bust. |
| `shiki.notify.psm1` | `Notify-Done` (BurntToast или MessageBox) — **в `shiki.main.ps1` сейчас не вызывается**; можно вызвать вручную после синхронизации. |

## Формат заметки

- Имя файла: `NNNNN - Название.md` (id + безопасное имя, обрезка длины пути).
- Front matter: `name`, `tags`, `shikimori_id`, `shikimori_url`, `cover`, статус, оценка, метаданные тайтла.
- Оценка «моя»: сначала ищется число в тексте комментария (`Get-ScoreFromComment`), иначе используется `score` из списка оценок.

## Кэш

В `shiki_cache` хранятся файлы `<id>.json`. Устаревшие по TTL (по умолчанию 336 ч) перезапрашиваются; записи для id, которых больше нет в текущем списке, удаляются.

## Прогресс в консоли

Скрипт выводит шаги **[1/8] … [8/8]**: старт с URL API (`SiteUrl`, обычно [shikimori.io](https://shikimori.io)), OAuth и профиль, список оценок, число уникальных тайтлов, затем самый долгий шаг — детали аниме. На шаге 5 показывается **полоса прогресса** PowerShell (`Write-Progress`) и подписи к блокам; отдельная строка на каждый `CACHE_OK` по умолчанию отключена (чтобы не забивать лог). Для отладки кэша можно вызвать `Get-ShikiDetailsBatched` с `-VerboseCacheLog` из своего сценария или временно добавить флаг в коде.

## Устранение неполадок

- **429 / 503:** в `Invoke-JsonWithRetry` есть экспоненциальная задержка и учёт `Retry-After`.
- **В логе `TerminatingError(Invoke-RestMethod)` / «удалённый хост разорвал подключение» / таймаут:** сетевой сбой или обрыв до сервера; повторный запуск обычно помогает. Повторные попытки **не** делаются для всех сетевых ошибок (только для 429/503).
- **`API_ERROR: ID=…` в фазе деталей:** для этого id не удалось получить JSON; заметка для него может быть пропущена или частичная. Запустите синхронизацию ещё раз после стабилизации сети.
- **Нет постеров:** проверьте `StaticBase`/`Base`, опционально `ShikiCookie`.
- **Ошибка на Windows PowerShell 5.1:** запускайте через `pwsh` из PowerShell 7.
